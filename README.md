# Taco Bell Greece — Sales & Food Cost Dashboards

Internal dashboards for Taco Bell Greece (operated by FRT).

## 🌐 Live URLs

- **Sales Dashboard:** https://reports-frt.github.io/tacobell-sales-dashboard/
- **Food Cost Hub:**  https://reports-frt.github.io/tacobell-sales-dashboard/food/

Password protected (same as KFC dashboard).

## 📂 Structure

```
tacobell-sales-dashboard/
├── index.html                          # Sales dashboard (main)
├── data.json                           # Sales data (auto-generated by pipeline)
├── food/
│   ├── index.html                      # Food Cost Hub
│   └── food_data.json                  # Food cost data (auto-generated)
│
├── automation/
│   ├── update_dashboard.py             # Sales pipeline: Outlook → parse → push
│   ├── run_update.bat                  # Daily runner for Sales pipeline (12:00)
│   ├── setup_task.ps1                  # Setup scheduled task (Sales)
│   │
│   └── foodcost/
│       ├── pull_food_emails.py         # Food: pull TB emails from Outlook
│       ├── build_food_data.py          # Food: parse xlsx → food_data.json
│       ├── build_pipeline.py           # Food: orchestrator (pull + build + push)
│       ├── run_daily_food.bat          # Daily runner for Food pipeline (12:30)
│       ├── build_food_data.bat         # Manual run (no pull, just rebuild)
│       └── setup_food_task.ps1         # Setup scheduled task (Food)
│
└── _work/                              # Working directory (gitignored)
    ├── *.xlsx                          # Downloaded source files
    └── *.log                           # Pipeline logs
```

## 🔁 Brand isolation

This pipeline **only** picks up emails whose subject contains `tb` or `taco`
(case-insensitive, anywhere in subject). KFC emails (without TB markers) are
ignored.

The companion KFC repo (`kfc-sales-dashboard`) does the inverse: it rejects
any subject containing `tb` or `taco`.

## ⏰ Scheduled tasks

| Pipeline | Schedule | Source emails (subject must contain "tb"/"taco") |
|----------|----------|--------------------------------------------------|
| Sales    | 12:00    | `Weekly_Report_New_TB`, `Time_and_Day_TB`         |
| Food     | 12:30    | `Μεικτό κέρδος_TB`, `Πωλήσεις_*_TB`               |

Both run **after** the KFC pipelines (KFC sales at 11:00, KFC food at 11:00)
to avoid Outlook contention.

## 🔧 Initial setup (one-time)

```powershell
# 1. Clone repo
cd C:\Users\IT\Documents\GitHub
git clone https://github.com/Reports-frt/tacobell-sales-dashboard.git
cd tacobell-sales-dashboard

# 2. Verify Python is installed (3.10+)
python --version

# 3. Install dependencies (same as KFC repo)
pip install pywin32 openpyxl pandas

# 4. Setup scheduled tasks
cd automation
.\setup_task.ps1
cd foodcost
.\setup_food_task.ps1

# 5. Manual test run (after Targit reports are configured)
cd ..
.\run_update.bat
```

## 🐛 Troubleshooting

- **No email found:** Check the email subject contains `tb` or `taco`.
  See logs in `_work/update.log`.
- **Outlook MAPI error:** Ensure Outlook is running before the scheduled task fires.
- **Push failed:** Check `git status`. May need to pull first.
